  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Porque si no se hace, el servidor puede recibir datos que no son válidos y eso puede causar problemas.

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    Desactivando JavaScript en el navegador, o usando un navegador que no soporte JavaScript.

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Porque aunque se pueda saltar la validación, es mejor que el servidor no reciba datos que no son válidos. De igual manera nos ayuda a mejorar la experienia del usuario dandole feedback si realizó algo mal antes de enviar los datos al servidos y que retorne un error desde el backend.
